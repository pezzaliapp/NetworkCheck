## 2. Obiettivo dell'app

NetworkCheck è una **web app / PWA educativa e diagnostica** per utenti non tecnici.
Aiuta a capire cosa è esposto nella propria rete locale e sul proprio computer.

- L'app **NON** esegue scansioni dal browser.
- Guida passo passo con i comandi corretti per **macOS / Windows / Linux**.
- Spiega cosa fa ogni comando e aiuta a interpretare l'output **incollato** dall'utente.
- **Tutta** l'analisi avviene **lato client**. Nessun dato lascia il dispositivo.

---

## 3. Principio di onestà (anti security-theater)

- Niente promesse di sicurezza. L'app non sostituisce un audit né fa scansioni reali.
- L'interprete dell'output è **euristico**: usa sempre linguaggio probabilistico
  ("possibile", "probabile", "da verificare"). **Mai** verdetti tipo "sei al sicuro".

---

## 4. Tecnologie (zero dipendenze)

- HTML5, CSS3, JavaScript vanilla (ES modules).
- **Nessun** framework, libreria CDN, font esterno, tracking, analytics, backend o cloud.
- PWA installabile, offline dopo il primo caricamento.
- **Nessuna** chiamata di rete in uscita a runtime (DevTools → Network deve restare vuoto
  dopo il load). Font di sistema (`system-ui` stack).
- Tutti i percorsi **relativi** (`./`) per funzionare su GitHub Pages in sottocartella.

---

## 5. Hardening dell'app (ciò che cerca un critico)

1. **CSP** via `<meta http-equiv>`: `default-src 'self'`; niente `'unsafe-inline'` per gli
   script (tutto il JS in file esterni); niente `eval` / `new Function`.
2. **Anti-XSS:** l'output incollato è UNTRUSTED. Inserisci risultati solo con
   `textContent` / `createElement`, **mai** `innerHTML` su dati utente.
3. **Anti-ReDoS:** regex semplici e lineari. Limita l'input (~100 KB) con messaggio chiaro.
4. **Privacy storage:** salva solo preferenza lingua e consenso al disclaimer; mai i
   contenuti incollati. Pulsante "Cancella tutto".
5. **Service worker:** cache versionata, cache-first con fallback offline, pulizia cache
   vecchie all'attivazione.

## 6. Disclaimer / gate di consenso (primo avvio)

Modale bloccante accessibile (focus trap, ruoli ARIA `dialog`):
- uso personale, educativo, diagnostico;
- comandi di scansione (nmap) solo su reti/dispositivi propri o autorizzati — scansionare
  reti altrui può costituire reato;
- nessun dato raccolto/inviato/memorizzato esternamente (coerente con GDPR).

Checkbox "Ho letto e accetto" + "Entra". Consenso ricordato. Link al disclaimer nel footer.
